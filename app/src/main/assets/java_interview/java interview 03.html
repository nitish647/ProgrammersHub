Answer: In the Just-in-Time compilation, the required code is executed at run time. Typically, it involves translating bytecode into machine code and then executing it

directly. For enabling high performance, Java can make use of the Just-In-Time

compilation. The JIT compiler is enabled by default in Java and gets activated as

soon as a method is called. It then compiles the bytecode of the Java method into

native machine code. Thereafter, the JVM calls the compiled code directly instead of

interpreting it. This grants a performance boost.
